1. What “Python Syntax” Really Means

When people say Python syntax, they simply mean:

The rules Python follows to understand your instructions.

Just as English has grammar rules, Python has syntax rules. The good news is that Python’s syntax is simpler and more readable than most programming languages.

Key Ideas to Remember

  • Python reads code from top to bottom
  • Python is case-sensitive
  • Python cares about spacing and indentation
  • Python executes instructions exactly as written

Python does not guess what you mean.

3. Numbers in Python

Integers (Whole Numbers)

cases = 120
deaths = 5

Integers are used when:

  • Counting people
  • Counting events
  • Working with totals

Floating-Point Numbers (Decimals)

rate = 2.5
average_age = 34.7

Floats are used when:

  • Precision matters
  • You are calculating averages or rates
  • You expect decimal results

Checking the Type of a Number

type(cases)
type(rate)

You should see:

  • int for integers
  • float for decimals

4. Strings: Working with Text

A string is any text enclosed in quotation marks. 

country = "Nigeria"
program = "HIV Prevention"

Python accepts:

  • Double quotes "text"
  • Single quotes 'text'

Be consistent.

Printing Strings

print(country)
print(program)

Combining Text (Very Gently)

print("Country:", country)

Python prints:

Country: Nigeria

Common Beginner Mistake

❌ Forgetting quotation marks:

country = Nigeria

✅ Correct:

country = "Nigeria"

5. Booleans: True or False

is_active = True
is_complete = False

Important rules:

  • Capital T in True
  • Capital F in False

Booleans are critical later for:

  • Filtering data
  • Writing conditions
  • Making decisions in code
Try This Yourself
  • Create one integer
  • Create one float
  • Create one string
  • Create one boolean

Print all four.

6. The print() Function (More Carefully)

print("Total cases:", cases)

You can print:

  • Text
  • Variables
  • Results of calculations
print(10 + 5)

Python prints:

15

Why print() Matters

Without print(), Python often calculates silently. As a beginner, always print results so you can see what is happening.

7. Comments: Writing Code for Humans

# Store the number of reported cases
cases = 120

Python ignores comments completely.

Why Comments Matter in Real Projects

  • You will forget what code does
  • Other people will read your code
  • Clear explanations build trust

9. Common Beginner Errors (And Why They Happen)

❌ Missing parentheses:

print "Hello"

✅ Correct:

print("Hello")

❌ Misspelling function names:

pritn("Hello")

❌ Mixing text and numbers incorrectly:

print("Total:", 10 + "5")

Text and numbers must be handled carefully.
