1. Why Functions Exist (The Big Idea)

Before functions, let’s revisit a simple problem. 

values = [10, 20, 30]
average = sum(values) / len(values)
print(average)

Now imagine doing this 20 times, in different places, with different lists.

You would repeat the same logic again and again.

This is exactly the problem functions solve.

A function is a named block of code that performs a specific task and can be reused.

Functions help you:

  • Avoid repetition
  • Organize logic
  • Reduce errors
  • Make code readable and trustworthy

In professional analytics, functions are not optional.

2. Built-in Functions: Functions You Already Use

You have already used functions—even if you did not call them that.

Example: print()

print("Hello, world")

Here:

  • print is a function
  • It performs an action
  • It takes an input (what to print)

Other Built-in Functions

len([1, 2, 3])
type(10)
sum([5, 10, 15])

Python ships with many built-in functions so you don’t have to reinvent the wheel.

Try This Yourself 
numbers = [4, 8, 15, 16, 23, 42]

print(len(numbers))
print(sum(numbers))
print(type(numbers))

3. Writing Your Own Function (Step by Step)

The Basic Structure

def function_name():
    # code goes here
  • def means define a function
  • You choose the function name
  • Parentheses will later hold inputs
  • Indentation matters

A Simple Function with No Inputs

def greet():
    print("Welcome to Python!")

To use the function: 

greet()

Nothing happens until you call the function.

4. Functions with Inputs (Arguments)

def greet_person(name):
    print("Welcome,", name)

Call it like this: 

greet_person("Amina")
greet_person("David")

Here:

  • name is a parameter
  • "Amina" is an argument

Functions allow you to write logic once and reuse it many times.

5. Returning Values (Not Just Printing)

Printing shows results. Returning gives results back to the program.

Example

def calculate_average(values):
    return sum(values) / len(values)
scores = [70, 85, 90]
avg_score = calculate_average(scores)
print(avg_score)

Key Difference

  • print() → displays output
  • return → sends output back for reuse

Common Beginner Mistake

def calculate_average(values):
    sum(values) / len(values)  # No return

Python calculates but returns nothing.

6. What Are Methods? (Functions Attached to Objects)

This distinction is subtle—but important.

Functions vs Methods

  • Function → stands alone
  • Method → belongs to an object
text = "python"
print(text.upper())

Here, upper() is a method belonging to the string text.

Common String Methods

text.upper()
text.lower()
text.title()
country = "nigeria"
print(country.title())

7. Why Methods Matter in Data Analysis

When working with data:

  • Columns have methods
  • DataFrames have methods
  • Methods make transformations readable

You will see this heavily with Pandas.

8. Introducing Libraries (Carefully)

Python is powerful because of libraries.

A library is a collection of pre-written code for a specific purpose.

Importing a Library

import math
math.sqrt(25)

Here:

  • math is the library
  • sqrt is a function inside it

9. The Three Core Libraries for This Book

We will work extensively with:

  • NumPy — numerical operations
  • Pandas — data analysis and manipulation
  • Plotly — interactive visualization

Importing Them (Standard Practice)

import numpy as np
import pandas as pd
import plotly.express as px

These aliases (np, pd, px) are industry standards.

10. First Gentle Look at Pandas

import pandas as pd

data = {
    "country": ["Nigeria", "Kenya", "Ghana"],
    "cases": [1200, 850, 600]
}

df = pd.DataFrame(data)
print(df)

You just created a DataFrame—a table-like structure.

We will explore this deeply in the next chapters.

11. Using Libraries Responsibly

Libraries:

  • Save time
  • Reduce errors
  • Enable complex analysis

But you must:

  • Understand what they do
  • Read error messages
  • Avoid blind copying

Clarity always comes first.

12. Common Beginner Mistakes with Functions & Libraries

  • ❌ Forgetting parentheses (print instead of print())
  • ❌ Misspelling library aliases
  • ❌ Expecting libraries to “just work”

Libraries require correct syntax, inputs, and file paths.
