1. What Is Control Flow? (In Plain Language)

So far, Python has followed instructions line by line, without choice.

But real programs need to make decisions.

For example:

  • If cases are high, flag the region
  • If data is missing, skip the record
  • If a value meets a condition, calculate something

This is called control flow.

Control flow is how Python decides what to do next based on conditions.

Without control flow, Python would be little more than a calculator.

2. Conditional Statements: if, elif, else

The Basic Idea

A conditional statement allows Python to ask a question:

“If this condition is true, do this.”

A Simple Example

cases = 1200

if cases > 1000:
    print("High number of cases")

What is happening here?

  • cases > 1000 is a condition
  • Python checks if it is True
  • If yes, the indented code runs
  • If no, Python skips it

Important Rule: Indentation Matters

Python uses indentation (spaces) to know what belongs together.

if cases > 1000:
print("High number of cases")  # This will cause an error

Correct version:

if cases > 1000:
    print("High number of cases")
Try This Yourself

Change the value of cases and re-run the code. Observe when the message appears and when it does not.

3. Adding Alternatives with else

What if you want Python to do something when the condition is false? 

cases = 500

if cases > 1000:
    print("High number of cases")
else:
    print("Cases are manageable")

Python chooses one path only.

4. Multiple Conditions with elif

Real situations often have more than two possibilities. 

cases = 800

if cases > 1000:
    print("High burden")
elif cases > 500:
    print("Moderate burden")
else:
    print("Low burden")

Python evaluates conditions from top to bottom.

The first condition that is True wins.

Common Beginner Mistake

if cases > 500:
    print("Moderate")
elif cases > 1000:
    print("High")

The second condition will never run.

5. Comparison Operators (The Language of Conditions)

Operator Meaning Example
> Greater than cases > 1000
< Less than cases < 500
>= Greater or equal cases >= 1000
<= Less or equal cases <= 100
== Equal to country == "Nigeria"
!= Not equal status != "Closed"

Important Distinction

= assigns a value
== compares values

This is one of the most common beginner errors.

6. Logical Operators: Combining Conditions

Sometimes one condition is not enough.

Python allows you to combine conditions using:

  • and
  • or
  • not

Using and

cases = 1200
year = 2023

if cases > 1000 and year == 2023:
    print("High burden this year")

Both conditions must be true.

Using or

if cases > 1000 or year == 2023:
    print("Condition met")

Only one condition needs to be true.

Using not

is_reported = False

if not is_reported:
    print("Data not yet reported")
Try This Yourself

Write a condition that prints a message when:

  • Cases are above 500
  • And the country is Nigeria

7. Introducing Loops (Repeating Actions)

Loops allow Python to repeat tasks.

This is essential when working with datasets.

The for Loop (First Encounter)

countries = ["Nigeria", "Kenya", "Ghana"]

for country in countries:
    print(country)

What happens?

  • Python takes the first item
  • Assigns it to country
  • Runs the indented code
  • Moves to the next item
  • Repeats until finished

Why Loops Matter

Loops allow you to:

  • Process each row of data
  • Apply rules repeatedly
  • Scale analysis to large datasets

Without loops, data analysis would be impossible.

8. Applying Logic Inside a Loop

cases_list = [1200, 450, 800]

for cases in cases_list:
    if cases > 1000:
        print("High burden:", cases)
    else:
        print("Lower burden:", cases)

Python evaluates the condition for each value.

9. Applying Logic to Real Datasets (Conceptual Preview)

Later, you will work with Pandas DataFrames, but the logic is the same. 

for country in countries:
    if country == "Nigeria":
        print("Priority country")

Understanding logic now makes Pandas much easier later.

10. Common Beginner Mistakes with Control Flow

  • ❌ Forgetting indentation
  • ❌ Using = instead of ==
  • ❌ Writing unclear or overlapping conditions
  • ❌ Forgetting Python stops at the first true condition

These are learning mistakes — not failures.
