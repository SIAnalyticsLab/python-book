1. What Is Python? (In Plain Language)

Python is a programming language.

A programming language allows humans to give instructions to computers in a structured, readable way. Python is especially popular because:

  • It reads almost like English
  • It has a gentle learning curve
  • It is used across many fields
  • It has powerful libraries for data analysis

When we use Python for data analysis, we typically ask it to:

  • Read data files (CSV, Excel, databases)
  • Clean and prepare data
  • Perform calculations and summaries
  • Create charts and dashboards
  • Automate repetitive tasks

Python does not think. It follows instructions exactly as written.

That is why clarity matters.

2. Why We Start with Google Colab

What Is Google Colab?

Google Colab (short for Collaboratory) is a cloud-based Python environment that runs entirely in your web browser.

You do not need to:

  • Install Python
  • Configure software
  • Worry about your operating system

You simply open a browser and start coding.

Why Colab Is Ideal for Beginners

  • No installation issues
  • Works on Windows, macOS, Linux, or tablets
  • Code runs instantly
  • Easy to save and revisit work
  • Ideal for experimentation and learning

In professional settings, local installations matter — but not yet.

Try This Yourself — Open Google Colab
  1. Open your browser
  2. Go to: https://colab.research.google.com
  3. Click New Notebook
  4. You will see a blank notebook with a code cell

A notebook is simply a document that contains:

  • Code cells (where Python runs)
  • Text cells (for notes and explanations)

3. Your First Python Command

In the first code cell, type:

print("Hello, Python!")

Now press Shift + Enter.

What Just Happened?

  • print is a function
  • A function performs an action
  • The parentheses () tell Python what the function should work on
  • "Hello, Python!" is text, called a string

The print() function tells Python:

“Display this text on the screen.”

4. Understanding How Python Executes Code

Python executes code line by line, from top to bottom.

This means:

  • Order matters
  • You cannot use something before defining it
  • Errors stop execution immediately

This predictable behavior is a strength — not a weakness.

5. Variables: Storing Information

A variable stores information so you can use it later.

Think of a variable as a labeled container. 

age = 30

Here:

  • age is the variable name
  • = assigns a value
  • 30 is the value stored

Now try:

print(age)

Python will display:

30

Why Variables Matter

  • Reuse values
  • Avoid repetition
  • Make code readable
  • Update logic easily

7. Comments: Explaining Your Code

# This is a comment
years = 10  # Years of experience

Comments make code:

  • Correct
  • Readable
  • Explainable

8. Common Beginner Mistakes

❌ Forgetting quotation marks

name = John

✅ Correct:

name = "John"

❌ Running cells out of order

Always run notebooks top to bottom.

❌ Assuming Python understands intent

Python only understands exact instructions.

9. Using ChatGPT Responsibly While Learning Python

Good Uses

  • Ask for simpler explanations
  • Understand error messages
  • Request commented examples
  • Clarify unfamiliar concepts

Avoid

  • Copying code blindly
  • Skipping practice
  • Letting AI replace thinking

Good test: Can I explain what this code does in my own words?
